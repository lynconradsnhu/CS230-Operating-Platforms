The Gaming Room offers a gaming applications called Draw It or Lose It, which allows users to start sessions with others, creates teams, choose images and make guesses. Their application needed to be available across clients and platforms.

5. How did you interpret the user’s needs and implement them into your software design? Why is it so important to consider the user’s needs when designing?

I documented the requirements based on the client's needs and analysis and focused implementation on the business needs and the user's perspective. If the application doesn't work as expected for the end user, the development effort failed.

6. How did you approach designing software? What techniques or strategies would you use in the future to analyze and design a similar software application?

I start with an analysis of the business rules and functional and non functional requirements. I also decide on security practices and order of implementation. I use iterative development, checking my code frequently for mistakes or logical faults.
